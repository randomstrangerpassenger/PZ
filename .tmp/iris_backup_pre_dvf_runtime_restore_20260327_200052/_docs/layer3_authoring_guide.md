# 3계층 작성 가이드

**대상 파일**: `build/data/description/layer3_registry.json`

---

## 1. 레지스트리 구조

```json
{
  "version": 1,
  "entries": [
    {
      "fulltype": "Base.TinOpener",
      "status": "active",
      "kind": "distinct_use",
      "text_ko": "소지하고 있으면 통조림을 열 수 있다.",
      "anchors": ["uc.rightclick.can_open_canned_food"],
      "notes": "작성 메모 (비출력)"
    }
  ]
}
```

---

## 2. 필드별 작성법

### `fulltype`
- 게임 데이터의 정확한 FullType (예: `Base.Hammer`, `Base.TinOpener`)
- 존재하지 않는 fulltype은 T1 FAIL

### `status`
| 값 | 의미 |
|---|---|
| `active` | 3계층 문장을 출력한다 |
| `silent` | 의도적으로 3계층을 비운다 (1·2계층으로 충분할 때) |

### `kind`
문장의 성격을 분류한다. `status=active`일 때 필수, `status=silent`일 때 `null`.

| kind | 의미 | 예시 |
|---|---|---|
| `identity` | 아이템의 정체성 서술 | "이것은 응급처치 키트다." |
| `distinct_use` | 고유한 용도 서술 | "소지하고 있으면 통조림을 열 수 있다." |
| `constraint` | 사용 제약 서술 | "전기가 연결되어 있어야 작동한다." |
| `exception` | 예외적 속성 서술 | "물에 젖으면 사용할 수 없다." |
| `variant_note` | 변형/파생 관계 서술 | "오픈된 상태의 캔이다." |

### `text_ko`
- 자연스러운 한국어 서술문
- 종결 부호(`.`, `!`, `?`)로 끝나야 함
- 8자 이상, 120자 이하 권장

### `anchors`
- 선택적. 이 문장의 근거가 되는 증거 참조
- 형식: 문자열 배열. 예: `["uc.rightclick.can_open_canned_food"]`
- 비워도 됨: `[]`
- **주의**: `status=silent`이면 반드시 `[]`

### `notes`
- 비출력 메모. 작성 이유, 검토 이력 등 자유 기입

## 3. 허용/금지 문장 예시

### ✅ 허용

```
"소지하고 있으면 통조림을 열 수 있다."
"상처에 박힌 유리 파편이나 총알을 제거할 때 쓰인다."
"전기가 연결되어 있어야 작동한다."
"물에 젖으면 사용할 수 없다."
"오픈된 상태의 캔이다. 시간이 지나면 상한다."
```

### ❌ 금지

```
"이 아이템을 추천한다."                    → 추천 금지
"가장 좋은 근접 무기다."                   → 비교 금지
"효율이 좋다."                             → 해석 금지
"필수 아이템이다."                          → 판단 금지
"관련 레시피: 나무판자 만들기"              → 색인 헤더 (4계층 역할)
"Base.Hammer"                              → 내부 키 노출
"분류: 도구"                               → 내부 분류 노출
"무게: 2.0 / 데미지: 1.5 / 내구도: 15"     → 목록형 나열 (5계층 역할)
```

## 4. 침묵 작성법

1·2계층으로 용도 설명이 충분하여 3계층이 불필요한 경우:

```json
{
  "fulltype": "Base.SomeItem",
  "status": "silent",
  "kind": null,
  "text_ko": null,
  "anchors": [],
  "notes": "1·2계층으로 충분. 고유 설명 불필요."
}
```

또는, 레지스트리에 엔트리를 추가하지 않으면 **암묵 침묵**이 된다 (동일 효과).

---

## 5. 작성 우선순위

1. **고유 기믹이 있는 도구류**: 캔오프너, 핀셋, 렌치 등
2. **예외적 속성이 있는 아이템**: 물에 젖으면 못 쓰는 것, 전기 필요한 것
3. **변형/파생 관계가 있는 아이템**: 열린 캔, 반쯤 먹은 음식
4. **나머지**: 1·2계층으로 충분하면 과감히 침묵
