## 1. Objective

DVF 3-3 current authority chain에서 장기 보존되어야 하는 durable tracked surface와 round-local staging evidence / generated byproduct의 경계를 좁게 정의하고, 그 경계를 machine-readable inventory, VCS guard, required-validation manifest, claim-boundary docs로 봉인한다.

이 계획은 tracking 범위를 넓히는 계획이 아니라, 어떤 artifact가 clean checkout에서 반드시 보존되어야 하는지와 어떤 artifact가 staging evidence라는 이유만으로 durable surface가 되지 않는지를 분리하는 계획이다.

최대 claim은 다음으로 제한한다.

```text
DVF 3-3 durable current authority surface is narrowly classified.
Current-required durable paths are present, tracked, and not ignored.
Required evidence durability is tied to live required-validation adoption.
Tracking status is preservation evidence, not authority promotion.
No source, rendered, Lua bridge, runtime chunk, or package writer authority is opened.
```

이 계획의 완료는 source authority cutover, rendered regeneration, Lua bridge export, runtime chunk replacement, package payload mutation, release readiness, Workshop readiness, B42 readiness, deployment readiness, manual in-game QA, semantic quality completion, public-facing text acceptance, full clean-checkout required-evidence reproducibility, full historical byte reproducibility를 의미하지 않는다.

## 3. Non-Goals

이 계획은 다음을 해결하지 않는다.

* Durable surface를 authority surface와 동일시하지 않는다.
* VCS tracked status를 current authority status로 해석하지 않는다.
* ignored status를 non-important, disposable, deletion-approved status로 해석하지 않는다.
* required-validation manifest adoption을 source writer, rendered writer, Lua bridge writer, runtime writer, package writer로 취급하지 않는다.
* generated staging evidence root 전체를 current authority로 승격하지 않는다.
* non-required staging byproduct가 untracked라는 이유만으로 validation failure가 되게 하지 않는다.
* required-adopted evidence가 generated path 아래 있다는 이유로 durability requirement에서 빠지게 하지 않는다.
* candidate manifest를 live required-validation manifest처럼 소비하지 않는다.
* current-route runner closure guard를 우회하기 위해 current core closure나 tooling allowlist를 늘리지 않는다.
* `Iris/build/description/v2/output/dvf_3_3_rendered.json`의 final durable classification을 계획 단계에서 authority로 확정하지 않는다. 단, final closeout 전에는 explicit non-writer disposition을 확정해야 하며, unresolved `review_required` 상태로 final PASS를 내지 않는다.
* final closeout vocabulary를 release / runtime / package readiness로 확장하지 않는다.

## 9. Risk Analysis

### Architecture Risk

* Staging evidence root 전체를 durable surface로 잘못 승격하면 evidence preservation이 authority promotion처럼 보일 수 있다.
* Durable 경계를 너무 좁히면 clean checkout에서 required governance chain이 보존되지 않을 수 있다.
* Bounded durable-set sufficiency가 실패하면 taxonomy가 너무 좁거나 concrete backbone binding이 누락된 것이다.
* Required-adopted evidence와 non-required staging byproduct가 같은 root 아래 섞여 adoption 판정 오류가 발생할 수 있다.
* Current-route tooling allowlist를 새 durable guard 편의 확장면으로 오독할 수 있다.
* `output/dvf_3_3_rendered.json` disposition을 계획 단계에서 성급하게 final authority로 확정할 수 있다.
* `dvf_3_3_input_manifest.json`의 regeneration-manifest 성격을 second primary role로 처리하면 role matrix duplicate failure가 발생할 수 있다.
* Runtime chunk count를 hardcode하면 future chunk count 변화에서 false failure 또는 누락이 발생할 수 있다.

### Runtime Risk

* Runtime mutation은 계획 범위 밖이므로 직접 런타임 위험은 낮다.
* 잘못된 final wording이 runtime readiness나 package readiness로 오독될 위험은 있다.

### Compatibility Risk

* Required durable path가 untracked / ignored / missing이면 validation workflow가 새로 fail-loud할 수 있다.
* Required durable path가 tracked 상태라도 ignore-rule matched이면 fail-loud할 수 있다.
* Path separator normalization이 부족하면 Windows / POSIX에서 guard 결과가 달라질 수 있다.
* Current-route closure blocker가 새 test/tool import shape와 충돌할 수 있다.

### Regression Risk

* Existing required-validation manifest entries를 수정하거나 제거하면 current-route governance regression이 발생한다.
* Existing VCS policy guard와 새 durable guard의 classification vocabulary가 충돌할 수 있다.
* Candidate manifest를 live manifest처럼 소비하면 runner가 wrong manifest를 소비할 수 있다.
* Manifest adoption, artifact materialization, current-route validation 순서가 뒤집히면 required artifact missing failure가 순환적으로 발생할 수 있다.
* Final artifact hash report가 자기 자신을 해시 대상으로 삼아 false mismatch를 만들 수 있다.
* Inclusion-level sufficiency가 empirical clean-checkout reproduction으로 과대 해석될 수 있다.
