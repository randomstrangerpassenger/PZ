# Registry Authority Protected Surface Policy

The six current DVF inputs, current rendered outputs, exporter/runtime bridge, runtime chunks, public require/bootstrap files, and `Iris/build/package` are read-only protected identity surfaces for this closure.

Preflight, Execution Entry, implementation freeze, final rerun, and terminal verification use one closed denominator. Directories are hashed from sorted repo-relative child paths and raw SHA-256 values. Windows extended-length paths are enumerated without omitting files beyond the legacy MAX_PATH boundary.

Candidate, package-probe, receipt-fixture, and report writes remain inside the registered attempt evidence root. A protected mutation is a blocker and cannot be waived by a review or owner seal.
