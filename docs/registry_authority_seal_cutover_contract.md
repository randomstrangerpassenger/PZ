# Registry Authority Seal / Cutover Contract

Candidate identity is the closed hash set of current source bindings, rendered candidate, exporter receipt, runtime chunk candidate, and isolated package projection.

Promotion requires complete role classification, one current identity chain, fresh required evidence, zero protected dirty overlap, stale-guard PASS, and complete external review inputs. This governance closure does not perform a live cutover and issues no production receipt.

WP-5 is the sole issuer of contained `fixture_only` receipts. Each receipt binds the exact six inputs, normalized body-plan authority, candidate raw and canonical-entry hashes, target preimages and expected outputs, expiry, nonce, and an exclusive consumption marker. Replay, partial state, dual current, drift, or postcondition mismatch is rejected. Rollback snapshots are historical-only and never automatically reenter current authority.

Atomic live apply/restore is not verified here and requires a separate operational plan.
